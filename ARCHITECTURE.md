# Generative UI Mobile App — Architecture

## What we're building

A React Native (Expo SDK 54) app where the entire screen is generated by an LLM at
runtime. The user types a prompt into a bar fixed at the bottom. The model responds not
with plain text, but with a **UI description in a custom DSL**. The app parses that DSL as
a stream and renders it into real native components, top-to-bottom, as tokens arrive.

Components are interactive: tapping a card can push a new generated screen or open one as a
modal sheet, toggle/edit local state, submit data back to the model, deep-link into another
app, etc. Every screen is generated the same way — only the starting prompt differs.

The novel part is the DSL format (below), which makes streaming + nesting + progressive
rendering fall out for free.

> Stack: Expo SDK 54, React Native 0.81, React 19.1, TypeScript, bun. reanimated 4 (all
> animation), react-native-screens native-stack, react-native-svg (loading glow),
> react-native-keyboard-controller, @react-native-segmented-control, expo-image,
> zustand (+persist). Package manager and test runner: bun.
>
> The App Store doesn't support Expo SDK 57 — SDK 54 is the latest it supports, so we're on 54.

---

## Core principles (do not violate)

1. **Model produces intent (data), client produces behavior (code).** The model never
   sends functions — only action *descriptors*. The client owns every real handler.
2. **Render tree is ephemeral, state is persistent.** Every turn regenerates the tree.
   State (selections, toggles, filters, editable lists) lives in a separate store keyed by
   *semantic* IDs, so it survives regeneration and re-ordering.
3. **Small closed primitive pool, infinite composition.** ~15 action primitives bound to
   real native code. Complex behaviors come from *chaining* primitives, never from adding
   new ones. Business logic never becomes an action.
4. **Store is source of truth. Model reads state + writes it (whitelisted). User intent
   beats model assumption.**
5. **Screens replace, they don't append.** New turn = new screen pushed on the native
   stack. Old screens stay mounted/cached so Back is instant and free (no regeneration).
6. **Lazy + local-first.** First response returns structure only; slots fill as they
   stream. Edits that don't need fresh content (add to a list, toggle, switch filter) are
   pure local state — no round-trip.
7. **Atomic components, the AI composes.** No bespoke composite widgets (no "FilterSheet").
   Layout is built from atomic primitives (Row/Column/Carousel/Card); the model arranges
   them. A "filter in a bottom sheet" is `present()` of a chooser screen, not a component.

## The DSL

The model outputs a flat list of assignments. Nesting is done by *reference*, not by
physically nesting brackets. Every line is complete (streamable like JSONL) yet the tree
can be arbitrarily deep (like JSON).

### Grammar

```
program    := line+
line       := IDENT "=" expr
expr       := constructor | array | object | string | number | ref
constructor:= IDENT "(" args? ")"
args       := expr ("," expr)*
array      := "[" (expr ("," expr)*)? "]"
object     := "{" (pair ("," pair)*)? "}"
pair       := IDENT ":" expr
ref        := IDENT            // a bare identifier = reference to another line
string     := "..."           // double-quoted
```

- `root` is the mandatory entry point. Rendering starts there.
- A bare identifier inside args (e.g. `hero`, `r1`) is a **reference** to another line.
- **Forward references are expected.** `root` references `hero` before `hero` is defined.
  Until the referenced line arrives, its slot renders a **skeleton**.
- This is the "top-down fill" UX: `root` arrives first → shape known, N skeleton slots laid
  out → each subsequent line fills one slot.
- Extra well-known lines are metadata, not rendered (only what `root` reaches renders).

### Example (current components)

```
root  = Column([head, hero, chips, listcard, cta])
head  = CardHeader("Dinner near you", "Top-rated spots · 15–30 min delivery")
hero  = ImageBlock("https://picsum.photos/seed/burger/800/440")
chips = Chips(["All","American","Sushi","Thai"], "filter.cuisine", "All", "list")
listcard = Card([list])
list  = ListBlock([r1, r2, r3], "Nearby")
r1    = ListItem("The Daily Grill", "4.9 star · 15 min · $$", {src:"https://picsum.photos/seed/grill/200/200"}, Action([navigate("The Daily Grill menu with categorized dishes and prices")]))
r2    = ListItem("Sakura Zen", "4.7 star · 22 min · $$$", {src:"https://picsum.photos/seed/sushi/200/200"}, Action([present("Sakura Zen menu of nigiri and rolls with prices")]))
r3    = ListItem("Thai Basil", "4.6 star · 28 min · $$", {src:"https://picsum.photos/seed/thai/200/200"}, Action([navigate("Thai Basil menu with curries and noodles")]))
cta   = Buttons([order])
order = Button("Order Usual", Action([navigate("Usual order: pre-filled cart and Place Order button")]), "primary")
```

### Parser (`src/dsl/`)

- `DslStreamParser`: buffer tokens, split on newline, parse each complete line into a node,
  add to a live `SymbolTable` (`Map<name, Node>`), fire a callback each line so the screen
  grows incrementally. A trailing incomplete line is ignored until its newline arrives.
- `resolver.ts`: resolve `ref` nodes by name. Missing → skeleton. Cycles guarded (visited
  set + depth cap). `parseProgram(src)` parses a whole string at once (tests / fixtures).

## Component registry (`src/registry/`)

Constructor name → RN component + zod schema + arg extractors. The registry is the single
source of truth for what the model may emit; `catalog.ts` derives the component section of
the system prompt from it (never hand-written twice). Unknown constructor / failed schema /
cycle / too-deep → a safe `Fallback`, never a crash.

**Layout / containers** (`components/containers.tsx`)
| Constructor | Purpose |
|-------------|---------|
| Card        | vertical container with card chrome (surface, border, padding) |
| Column      | vertical stack, no chrome |
| Row         | horizontal — splits width EVENLY across children (equal cells) |
| Carousel    | full-bleed HORIZONTAL scroller (edge-to-edge); children keep own width |
| ListBlock   | titled vertical list (usually ListItem rows) |
| Buttons     | a Row specialised for a Button call-to-action bar |

**Media / content** (`components/media.tsx`, `content.tsx`)
| Constructor | Purpose |
|-------------|---------|
| ImageBlock  | hero/banner image (expo-image, lazy). Caption optional & rare |
| PhotoCard   | fixed-width photo card (image + title + subtitle), tappable — for Carousel |
| TextContent | text with a style variant (e.g. "large-heavy", "body") |
| CardHeader  | title + subtitle (use at the TOP of a screen) |
| KVList      | titled label/value pairs (details/summary) |
| HeroStat    | big value + label + caption |
| MapView     | static map placeholder pinned to an address |

**Interactive / stateful** (`components/interactive.tsx`)
| Constructor | Purpose |
|-------------|---------|
| ListItem    | tappable row: title, subtitle, thumbnail {src,alt}, Action |
| Button      | label + Action + variant ∈ {primary, secondary, ghost}; fills its cell |
| Chips       | horizontal selectable filter tabs; local-first; optional `region` for partial refine |
| StateList   | EDITABLE local-state collection (cart/list/to-do); built-in × remove |

Layout rules the prompt enforces: **header first** (title/subtitle at top, image below);
**captions rare** (don't repeat the title on the image); a component never sets its own
width — Row/Carousel/cells decide it; **horizontal scrollers (Chips, Carousel) must be
direct children of a Column root**, not inside a Card, so they bleed to the screen edge.

## Actions (`src/actions/`)

`Action([...])` holds a **sequential** list of primitive steps, run in order, no
conditionals. `dispatch(steps, node, ctx)`; a step may fall back to the owning node's first
arg (e.g. seed → node title). `catalog.ts` derives the action section of the system prompt.
Unknown primitive → no-op + warn.

### Primitive pool (closed)

| Primitive | Effect |
|-----------|--------|
| navigate(seed) | push a NEW full-screen generated screen |
| present(seed) | open a new generated screen as a MODAL SHEET over the current one |
| refine(prompt) | re-run generation for the WHOLE current screen in place |
| refineRegion(region, prompt) | regenerate ONLY one line + its children (partial refine) |
| setState(key, value) | write to the state store (whitelisted keys only) |
| add / remove / toggle(key, value) | LOCAL collection edits (StateList) — no regeneration |
| openUrl(url) | open a URL in the browser |
| openApp(app, query?) | deep-link into an external app (youtube/spotify/maps/twitter/instagram/appstore) |
| call(number) | real phone call (`tel:`), not a generated screen |
| share(text) / copy(text) | native share sheet / clipboard |
| submit(formKey) | collect `form.*` state under `formKey`, run a new generation with it |
| back() | React Navigation goBack |

No business-logic primitives (no `bookHotel`, `placeOrder`) — those are `submit` + a
backend. A primitive is a UI *mechanic*, never the *thing itself*.

## State system (`src/state/store.ts`)

A single Zustand store holds a flat "state bag": `semanticKey → value`.

- **Semantic keys, not positional**: `filter.cuisine`, `selection.destination`, `tab.menu`,
  `form.cart`. Stable across regeneration/reorder.
- **Local-first**: chip select, toggle, add/remove on a StateList mutate the store directly
  and re-render instantly — the model is never contacted.
- **Model reads state**: a compact snapshot is injected into the system prompt each turn.
- **Model writes state**: via `setState` (or a component's initial value), through a prefix
  whitelist `["selection.","filter.","expanded.","tab.","form."]`. Non-whitelisted → dropped.
- **Conflict rule**: user intent wins — a user-touched key is protected within a screen;
  `clearUserTouched()` on a genuinely new main-thread turn re-allows model writes.
- **Lifetime = window**: "New window" (`resetWindow` + store `reset`) wipes state + context.

## Navigation & history (`src/navigation/`, `src/screens/GenScreen.tsx`)

Native-stack (`@react-navigation/native-stack`). Routes: `Home`, `Gen` (pushed full
screen), `GenSheet` (modal), `Settings` (modal), `ModelPicker` (modal). `GenScreen` serves
both `Gen` and `GenSheet` (keyed by route name).

- `src/state/history.ts` maps `screenId → ScreenRecord { prompt, kind, table, status,
  context, loading[] }`. The native stack IS the history; Back pops to an already-mounted,
  cached tree — instant, no regeneration.
- **Replace model**: each turn `push`es a new route (`navigation.push`, never `navigate`,
  so presenting a sheet from within a sheet stacks a new modal instead of swapping params).
- **Detail screens are an isolated branch**: `navigate`/`present` create a `detail` record
  with minimal context (`Opened from: <prompt>`); they never append to the main-thread
  context chain, so opening cards doesn't pollute what the model sees on a later refine.
- **Modals**: presented with `presentation: 'modal'`. iOS hides the native back button on
  modals, so each modal supplies an explicit `headerLeft` X (close) → `goBack()`. Sheets
  are also swipe-to-dismiss (`gestureEnabled`). Modal screens force a light status bar (they
  sit over the dark dimmed backdrop, invisible with dark content in light theme).

## Rendering & animation (`src/render/`)

- `Renderer` starts at `root` and resolves refs recursively; nodes are memoized and stably
  keyed by resolved path so settled subtrees don't churn and animations don't restart.
- **Skeletons** for not-yet-streamed refs: block (default), button-height in a Row, and a
  two-part **PhotoCardSkeleton** (image + caption) in a Carousel.
- **Fallback** for unknown/invalid/cyclic/too-deep nodes — the app never crashes on garbage.
- **Entrance**: each column child fades + slides in (`enterFade`, a reanimated `entering`
  worklet) as its line arrives → staggered top-down fill. Height animates via
  `LinearTransition`. The region wrapper is `overflow: visible` (so a horizontal scroller
  child can bleed full-width without being clipped).
- **Partial refine**: `refineRegion` streams only the named line into a buffer, splices it
  into the live table (`mergeTable`), and overlays a **Shimmer** — a soft radial loading
  glow (react-native-svg, theme-neutral colour, fades in/out) — over just that region while
  the old content stays visible underneath.

## LLM integration (`src/llm/`)

- **Providers** (`providers.ts`): OpenRouter (default), OpenAI, Anthropic — user-selectable,
  a key per provider, a model per provider. OpenRouter also exposes some `:free` models.
  `openaiCompat.ts` (OpenRouter + OpenAI), `anthropic.ts`, shared SSE consumer (`sse.ts`)
  over `expo/fetch` streaming. `index.ts` (`streamDsl`) reads settings and picks the path.
- **System prompt** (`prompt.ts`) assembled at runtime from: (a) generated component
  catalog, (b) generated action catalog, (c) DSL format + few-shot examples, (d) current
  state snapshot, (e) context chain, plus the current **date/time** (so the model isn't
  blind to "today") and, for a partial refine, the target region.
- **Context chain** (`state/context.ts`): a light semantic summary of each produced tree is
  appended to the main thread — never the raw tree (token blowup). Detail branches keep
  their own minimal context.
- Stream tokens → parser → symbol table → renderer, live.

## Mock / demo (`src/llm/mock.ts`, `fixtures.ts`)

Mock mode drives the whole engine offline (parser → renderer → skeleton → actions → state)
with artificial per-line delay. It is **not** one static page: each default Home suggestion
has its **own baked screen** (authored once, what the model would return), and
`fixtureFor(prompt)` picks the match by a distinctive keyword; anything else (custom prompts,
detail seeds opened from a card) gets a generic detail **fallback**. This is a deliberate
demo-content exception to "no keyword routing" — with Mock OFF + a key, a suggestion
generates fresh from the real model, exactly as if typed.

---

## Theme & settings (`src/theme.ts`, `src/screens/SettingsScreen.tsx`, `state/settings.ts`)

- **Theme**: System / Light / Dark, `useTheme()` palette + text styles; `automatic`
  `userInterfaceStyle`. Components build styles via `useMemo(() => makeStyles(colors,text))`.
- **Settings** (persisted to AsyncStorage via zustand/persist): provider (native
  SegmentedControl), per-provider API key, model (ModelPicker — a pageSheet modal listing
  popular + free models), Mock mode toggle. Home shows the current provider/model status.
- **Prompt bar** (`ui/PromptBar.tsx`): fixed at the bottom (KeyboardStickyView from
  react-native-keyboard-controller). Submit continues in context; a "New window" button
  (hidden inside sheets) resets stack + context + state.
